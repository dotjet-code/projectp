# Figma プロンプト 01 — スタイルガイド（デザインシステム1枚）

このプロンプトを Figma 上の「Style Guide」フレーム生成に使う。以降の全画面プロンプト (02, 03, ...) はこのガイドの色・タイポ・罫線ルールを前提にする。

## プロジェクト文脈（プロンプト冒頭に貼る）

Project P は、12人が月次ステージで「バズ・配信・収支」の数字を競い、上位6名が主役 (PLAYER)、7位以下は PIT（敗者復活組）として次月の逆襲を狙う、競争型エンタメプロジェクトの Web サイトである。アイドル応援サイトではない。公営競技の出走表と、スポーツ新聞の一面紙面と、M-1グランプリ敗者復活戦の熱量を掛け合わせた「勝負の可視化」サイトを作る。

## 世界観の三層

1. **主幹**：公営競技（競艇）— 1〜6号艇の色記号、番号、出走表、データ主役
2. **骨格**：スポーツ新聞の組版（東スポ/競馬新聞/スポニチ一面）— 大見出し階層、ケイ線区切り、情報密度、赤ペン入稿の痕跡
3. **トーン（迫力と煽り）**：昭和のアングラポスター（横尾忠則、寺山修司、天井桟敷）+ G1プロレス興行ポスター + 少年誌見出し

グラフィックに漫画的キャラ化表現は使わない。顔写真は"ポスターの被写体"扱い — 大判だが、号艇色の斜め帯とゼッケン彫込みで印象を制御する。

## シグネチャ（ブランドの"顔"）

サイトを一瞬で ProjectP と認識させるための固定要素。全ページに最低 1 つは配置する。

### 1. 篆刻 P マーク（ブランドスタンプ）

- 朱肉判子風の「P」、`#D41E28` のベタ、正方外枠 `--signal-ink` の細線
- わずかに 2–4° 傾けて押す。縁はラフ（印影の欠けを残す）
- 使用場所: 全ページ左上 sticky（16×16px）、共有画像、フッタ、印刷物
- 既存のグラデーション円ロゴ（`from-primary to-primary-blue`）は**廃止**

### 2. 出走表マーク（セクション区切り）

- 1–6 号艇色の正方 6 マス横並び（各 `8×8px`）+ 下に黒ケイ `1px`
- 全ページのセクション区切りや SNS シェア画像に常設
- これを見ただけで "ProjectP だ" と分かるレベルの刷り込み

### 3. PASS LINE マーク

- 赤の横線 + 上下に ▲6 ▼7 の矢印ピクト + 中央に小さく `PASS` 文字
- 6位/7位境界の象徴、サイト全体のメタファー
- フッタ、About ページ、SNS プロフィールアイコンに使用可能

### 4. スタンプ判子システム（紙面オーバーレイ）

紙面の上に**朱肉判子でドンと押されたように**配置する漢字スタンプ。最大サイズ 120pt、1 画面 1 個まで。

基本語彙（固定、勝手に増やさない）:
- **速報** / **激震** / **号外** — 事件系
- **締切** / **解禁** — 時間系
- **首位** / **陥落** / **逆転** / **差し** — 順位系
- **勝** / **敗** / **決** — 結果系
- **待て** / **止め** — 制御系

判子の角度はランダム −8° 〜 +8° に自動揺らぎ。色は赤 `#D41E28` 基本、一部黒 `#111111`。縁は印影的にラフ化。

### 5. 手書き赤ペン痕

新聞紙面の上に**編集者が夜通し入れた赤ペンの痕**を置く。これが泥臭さの源。

- 丸印（大事な数字を囲む）、矢印（順位変動方向）、下線、短いコメント書き
- フォントは手書き風（`Klee One` or `Zen Kurenaido`）を細く使用、ペン先の震えを模す
- 赤は `#D41E28`、透明度 85%、わずかに紙面から浮くように
- 使う場所: 一面見出しの数字横、出走表の注目行、因縁カードの矢印
- 多用禁止。1 画面に 3 個まで。

### 6. オノマトペ活字

昭和プロレスポスター的な擬音・煽り語を**大きな活字スタンプ**で配置する（絵の吹き出しではない、活字の物量）。

- 「ドン。」「激闘。」「ウォォ。」「バチン。」「沸騰。」「静寂。」
- 和文見出しフォント（A1ゴシック B or 貂明朝 B）、48–88pt、ベタ塗り or 中抜き
- 色は `--signal-ink` 黒、または赤 `#D41E28`
- 必ず**句点つき**で使う（命令的に）

## モード設計

サイトには 2 モードがある。ページ単位で全反転しない。セクション単位で切り替える。

### STAGE モード（明 / 日常）

- トップ、メンバー一覧、プロフィール、暫定順位、予想、結果閲覧
- 新聞紙の地に黒組版。冷静。情報密度高め。

### RACE モード（暗 / 勝負）

- ライブ投票、敗者復活戦、月末特番告知、結果発表の演出ゾーン
- 夜の競技場照明。赤と黄の点描。没入。
- STAGE から RACE へはスクロールで「トンネル遷移」（背景が徐々に暗転、罫線が残光になる）で接続。

実装時は `data-mode="stage" | "race"` 属性で CSS 変数束を差し替える。

### 競艇 6 号艇色 — "節ごとの出走レーン"（アイデンティティではない）

重要: 号艇色はメンバー永続アイデンティティではなく、**節ごとに再アサインされる出走レーン**である。PLAYER 6 名のみ号艇 1–6 を持ち、PIT 6 名は号艇を持たない（後述のピットレーン扱い）。

| 号艇 | Hex | fg on color | 割当ルール |
|---|---|---|---|
| 1号 (白) | `#F5F5F0` / 枠線 `#111111` | `#111111` | 前節 1 位 |
| 2号 (黒) | `#1A1A1A` | `#F5F1E8` | 前節 2 位 |
| 3号 (赤) | `#D41E28` | `#FFFFFF` | 前節 3 位 |
| 4号 (青) | `#1E4BC8` | `#FFFFFF` | 前節 4 位 |
| 5号 (黄) | `#F2C81B` | `#111111` | 前節 5 位 |
| 6号 (緑) | `#0F8F4A` | `#FFFFFF` | 前節 6 位 |

PIT 枠（7–12 位）は「**ピットレーン**」として号艇色の代わりに `#4A5060` の無彩色プレート + 前節順位の小数字で表示。敗者復活の対象である視覚的記号として機能する。

各艇色トークンは `{ main, fg, bg-tint (20%), border, contrast }` の5値で展開。既存 `src/lib/projectp/boat-colors.ts` をこの体系に昇格し、さらに `resolveBoatColor(memberId, stageId)` のように節依存で解決する関数を用意する。

### パーソナル識別（永続、12 名分）

号艇色が節で変わるため、メンバー個人を識別する**永続マーク**を別途用意する。

- **ゼッケン番号 01–12**: 入場順の永続番号。Space Grotesk Bold の大数字。顔写真より先に目に入る。
- **パーソナルパターン**: 12 種の幾何パターン（ストライプ角度・ドット密度・格子ピッチの組合せ）を各メンバーに固定割当。カード縁、詳細ページ背景、SNSシェア画像に使用。
- **パーソナルカラーは使わない**（12 色は識別破綻する）。色の役割は号艇(節)とシグネチャ(機能)に限定し、個人識別はゼッケンとパターンに委ねる。

### 指標色（バズ / 配信 / 収支）

現行の cyan → blue → purple 系を「計器の色」として残す。メイン装飾からは外す。

| 指標 | Hex | 用途 |
|---|---|---|
| バズ | `#00BCFF` | グラフ線・バー |
| 配信 | `#1447E6` | グラフ線・バー |
| 収支 | `#7A3DFF` | グラフ線・バー |

## タイポグラフィ

### フォントスタック（実装想定）

- **欧文見出し（叫ぶ）**: `Anton` / `Unbounded Black` / `Bebas Neue`（OSS、Druk の代替）
- **欧文数字（スコア・順位）**: `Space Grotesk Bold` / `JetBrains Mono Bold`
- **和文見出し**: `貂明朝 B` / `A1ゴシック B` / `FOT-筑紫A見出しミン`
- **和文本文**: `Noto Sans JP`（現行維持）
- **スタンプ漢字（煽り）**: `A-OTF UD新ゴ Pr6N H` または判子風のブロック体（筆文字は避ける）

Figma 上では Druk Wide Bold を比較用に置いてもよいが、本番フォントはOSSで成立させる。

### 階層（5段）

| 段 | サイズ | 用途 |
|---|---|---|
| **ポスター見出し** | 和文 120–200pt / 欧文 160–240pt | 結果発表・月末特番告知・敗者復活ヒーローでのみ使う |
| 大見出し | 和文 64–88pt / 欧文 88–120pt | 一面紙面トップ、ページ見出し |
| 中見出し | 24–32pt | セクション見出し |
| 小見出し | 14–18pt | カード内タイトル、副見出し |
| キャプション | 10–12pt | 日付、脚注、ラベル |

数字（ポイント、順位、票数）は常に欧文数字フォント（Space Grotesk or JetBrains Mono）、本文より一段大きく置く。数字が主役。

### 多段重ね組版（迫力の作り方）

大見出し以上は**必ず 2–4 段で構成**し、単発の大文字にしない。単発はただデカいだけで迫力にならない。

構成例:
```
[速報]              ← 赤スタンプ 14pt
首位、              ← 和文 72pt 黒
  陥落。            ← 和文 88pt 黒、右に 40px ズラし、2px 赤版ズレ
3時間前、塩見きら ── 阿久津真央。差 8pt  ← サブ 16pt
```

- 2 段目と 3 段目は**意図的に右/左へオフセット**（30–80px）、左右非対称にする
- **版ズレ**: 黒テキストの背後に同テキストを赤 `#D41E28` で 2–3px ズラし、50–70% opacity で重ねる。印刷ミス感
- 和文と欧文を**段で使い分ける**: 見出し本体は和文明朝/ゴシック、日付や数字は欧文 Anton/Unbounded

### 混植の版面ルール

- 和文明朝（貂明朝 B）× 欧文 Anton = 紙面のトップ見出し専用
- 和文ゴシック（A1 B）× 欧文 Unbounded = セクション見出し・カードタイトル
- ひらがな・カタカナは明朝・ゴシックともに**天地 85% の縮小混植**で（漢字を立てる）

## グリッド & 余白

- 基準単位 `4px`
- コンテンツ最大幅 `960px`（既存踏襲）
- 外側余白 デスクトップ `24px` / モバイル `16px`
- セクション間 `48–80px`
- カード内パディング `20–24px`

**新聞組版の導入**: カードではなく**ケイ線（1px 実線 `--rule`）で区切る**。影・角丸は極力使わない。どうしても箱にするときは `border-radius: 4px` まで、影なし。

## 罫線・網点・装飾ルール

### 罫線

- 主区切り: `1px solid --rule`
- 副区切り: `1px solid --rule-soft`
- 強調区切り: `3px solid --signal-red`（見出し直下、勝負ゾーン入口）

### 網点（ハーフトーン）

- 面として敷かない。**強調ブロックの背景のみ**に 30% opacity で適用。
- ドット径 `4px`、ピッチ `8px`、角度 `45°`
- 実装は SVG filter / CSS `background-image: radial-gradient(...)` をユーティリティ化。

### 斜めストライプ

- 締切ゾーン・警告ゾーン・ポスター背景。`--signal-yellow` と `#111111` の 45° ストライプ、幅 `8px`。
- ポスター見出しの背景にベタで敷くのは**月末特番告知・結果発表のみ**許可。

### ベタ塗り面積（迫力の素材）

- **黒ベタ帯**: セクション見出しの背景に全幅の黒 `--signal-ink` 帯（高さ 80–160px）を許可。白抜き見出しをその中に置く。
- **黄ベタブロック**: ハイライトしたい 1 ブロックを `--signal-yellow` で全塗り、中の文字は黒。使用面積は 1 画面で最大 **20%** まで拡張（旧規定の 5% を緩和）。
- **赤ベタ**: 速報・勝者発表の帯に `--signal-red` 全幅、白抜き文字。

ベタ塗りは「静かな紙面の中の一撃」として置く。散らさない。

### 印刷ズレ表現

- 大見出しのロード時のみ、赤版が 1-2px ズレて 200ms で整列するマイクロモーション。日常のUIには使わない。

### スタンプ漢字

- 赤または黒のブロック体で「激」「熱」「逆」「決」「差」「逃」「追」を判子風の枠付きで配置。
- 最大サイズ 80pt、使用は 1 画面に 1 個まで。散らさない。

### ステッカー / マステ

- 使わない（Y2K ポップに寄りすぎる）。代わりに**切り取り線（破線）と新聞の日付スタンプ**を使う。

## 写真ディレクション

顔写真はそのまま使わず、**誌面処理**を必ず通す。

- **処理**: モノクロ or デュオトーン（`--fg` + `--bg`）、粒子ノイズ薄くのせ、コントラスト強め。元のアイドル的ライティングを消す。
- **トリミング**: バストアップ基本、号艇プレートと並ぶ正方 `1:1` + 全身切抜き `3:4` の二種。顔だけの切り抜きは使わない。
- **切抜き輪郭**: 直角クロップのみ。丸トリミングはランキング 1 位の特別演出のみで使用。
- **下撮り指針**（将来の再撮影用）: スタジオ無地背景、左上からの強い単光源、汗・動き感を許容、スマイルより真顔優先。ランニング動作・手を挙げるポーズ・カメラ直視の三種を基本。

## アイコン体系

- **基本**: 単線ストローク `1.5px`、`24×24px` グリッド、角半径 `2px`
- **スタイル**: 建築製図ライン風、装飾なし。ラウンドアイコンや slashed zero は禁止。
- **主要**: arrow-right / arrow-up-right / arrow-trend-up / arrow-trend-down / pause / play / share / external-link / chevron / x-close / check / clock / megaphone(速報) / flag(節) / whistle(締切)
- **金/銀/銅メダル**は使わない。順位は数字だけで表現する（メダル絵文字の代わりに「#1」「#2」「#3」または「首位」「2 位」「3 位」）
- 実装: SVG インライン、`fill="none" stroke="currentColor"` で色は親継承

## 状態設計

画面テンプレートの前提として、以下 6 状態をデザイン時に必ず並べる。

| 状態 | 見た目の原則 |
|---|---|
| Loading | スケルトンは角丸なしの灰色矩形（`--rule-soft` 塗り）、50% 幅、0.8s パルス。スピナーは使わない。 |
| Empty | 「データなし」ではなく**新聞の"休刊"スタンプ**: 「**— 本日は事件なし —**」を斜めストライプ枠に入れる |
| Error | 「**速報: 通信障害**」赤スタンプ + 再読込ボタン。404 エラーではなく"号外未達"の擬態。 |
| Offline | 「**ライブ中断**」斜めストライプ黄、帯で画面上端に常駐 |
| Maintenance | 紙面を横罫線で埋め尽くし「**休刊**」大文字 |
| No-data (ステージ未開催) | 「**次節開幕まで X 日**」のカウントダウンのみ、空白を隠さない |

## サウンドデザイン

デフォルトミュート。ユーザーが有効化したときのみ鳴る。

- **投票解禁**: 短い電子ブザー 200ms（電光掲示板の勝利音風、金属的）
- **締切到達**: 競艇のファンファーレを模した下降 2 音
- **順位変動**: サブリミナル級 8bit クリック、音量 -18dB
- **RACE モード突入**: 低音のドローン 600ms

音は一切自動再生しない。Web Audio API、`<audio>` は使わない。

## データ整合性と公平性

デザイン仕様として明示:

- **投票はログイン必須**（匿名禁止）。ログイン前の投票UIは閲覧のみ。
- **多重投票対策**: 会員 ID + デバイスフィンガープリント + Cloudflare Turnstile。同一ユーザーの再投票は締切まで書換（新投票ではない）。
- **結果の裏付け**: スコア算定式をフッタ常設「**採点規程**」リンクで公開。透明性でガチ感を担保。
- **改竄防止の見た目**: 数字には常に**最終更新時刻**を `12:34:08 JST` 形式で併記。時刻が動いてるだけで信頼が乗る。

## コンポーネント・プリミティブ

### 1. 号艇バッジ (BoatBadge)

- 正方形 `32×32px`（小） / `48×48px`（中） / `72×72px`（大）
- 号艇色ベタ塗り、中央に白か黒のアラビア数字（その節での号艇番号 1-6）
- 1号は白地黒数字、枠線 `1px #111`
- **PIT 版**（号艇なし）: `#4A5060` 塗り、中央に「P」+ 下に小さく前節順位（例「#8」）

### 0. ゼッケン (JerseyNumber)

- 正方形 `40×40` 〜 `80×80`、白地に黒数字 `01`〜`12`
- Space Grotesk Bold、2桁ゼロパディング
- 号艇バッジと左右に並べて使うのが基本配置

### 2. 順位ナンバー (RankNumber)

- Space Grotesk Bold、サイズ 64–120pt
- 暫定順位は「#」プレフィックスなし、ただ大きな数字。
- 順位変動表示は右肩に小さな `▲2` / `▼1` / `−` を赤/青/黒で。

### 3. スコアバー (ScoreRow)

- 横棒 `height: 6px`、地色 `--rule-soft`、棒色は指標色
- 右端に数字（Space Grotesk Bold）、棒と同じ基線で整列
- カード内 3 段積み（バズ/配信/収支）

### 4. メンバーカード（出走表レコード）

横長レコード形式。左から:

- **ゼッケン (永続 01-12)** `40px`
- **号艇バッジ or PIT プレート (節依存 1-6 or P)** `48px`
- 顔写真（正方 `56×56`、デュオトーン処理済み）。丸トリミングは禁止、直角クロップ
- 氏名（和文見出し 18pt）+ ロール（PLAYER / PIT）小バッジ + ライブ中インジケータ（赤点 6px）
- スコアバー 3 段（バズ / 配信 / 収支）
- 今節ポイント合計（大数字 32pt）+ 変動（▲▼1 行）
- 右端に「詳細 →」テキストリンク（アイコンではなく文字）

角丸・影なし。上下に `1px --rule` の罫線のみ。背景にパーソナルパターンを 5% opacity で敷いても良い（個人識別の強化）。

### 5. 節タイムライン (StageTimeline)

トップ常設。4節 (開幕 / 中盤 / 終盤 / 特番) を横長の細帯で表示。現在位置を赤丸でマーク、進行率を黒バーで可視化。各節の日付を小キャプション。

### 6. ☆履歴バッジ (StageHistoryStar)

M-1の過去決勝☆を踏襲。過去ステージで PLAYER 入りした回数を ☆ で並列。7ステージ分まで横に並べ、該当は黒塗り、非該当は白抜き。

### 7. カウントダウン (RaceCountdown)

締切までの残時間。2 時間以内になると `--signal-red` + 斜めストライプ + 微振動モーション。日常は黒数字で静止。

## モーション原則

日常・節目・勝負の 3 段階で予算配分。

| 段 | 画面 | 動きの強度 |
|---|---|---|
| 日常 | トップ、順位閲覧、プロフィール | 静的。データ更新は 200ms でタイプライタ式カウント着地 |
| 節目 | 順位変動、新着速報 | 上昇矢印+色フラッシュ、1 回限り、600ms |
| 勝負 | ライブ投票、結果発表、月末特番 | フル演出。スタジアム光、紙吹雪、大見出しの印刷ズレ |

トランジション基調: `cubic-bezier(0.2, 0, 0, 1)`、300–500ms。

### 署名級モーション（迫力演出、固有名詞扱い）

一度見たら ProjectP と分かる、固定の演出セット。勝手に増やさない。

- **入場の判子演出** — 初回訪問時 / 節開幕日に、篆刻 P マークが画面中央で"ドン"と押される (300ms スケール 1.4→1.0、わずかに回転、0° で着地)。ページ背面から紙面が立ち上がる。
- **刷り込みセクション遷移** — セクション見出しが全幅の黒帯を一瞬で横切る (400ms、右から左)。インクが走る感覚。
- **フリップ順位** — 順位変動時、数字はデジタルカウントではなく**パタパタ式フリップ** (5 枚のコマ、各 60ms)。物理的。
- **インク流し込みトンネル** — STAGE → RACE 遷移は暗転グラデーションではなく、**画面端から黒インクが滲み進む** (SVG マスクで液体エッジ、600ms)。
- **紙吹雪 / 号外撒き** — 結果発表の瞬間のみ、画面上部から赤黒黄の紙片が 2s かけて舞い落ちる。`canvas` or DOM 要素、30 枚まで。
- **スタンプ連打** — 月末特番告知の開幕時、判子スタンプが 4 つ (「激震」「決戦」「頂上」「待て」) が 100ms 間隔で画面の四隅に押される。

全て `prefers-reduced-motion` で無効化。サウンド連動はユーザー有効化時のみ。

## コピーライティングのルール

### NG 語（アイドル化の罠）

推し / 応援 / 好き / ときめき / ワクワク / かわいい / 主役（詩的文脈での） / きらめき / 輝く / 夢 / 絆

### OK 語（競技語彙）

予想 / 賭け / 支持 / 見立て / 差し / 逃げ / 追走 / 捲り / 首位 / 陥落 / 上げる / 落ちる / 逆転 / 粘る / 伸びる / 食らう / 届く

### 見出しの型

- 動詞 + 数字 + 固有名詞
- 句読点で叩き切る
- 例: 「首位陥落。」「逆襲のPIT。」「ラスト72時間。」「7位、上げる。」「3ポイント差、届くか。」

### セクション見出し

漢字 + 数字主体。「本日の出走」「暫定順位」「逆転ライン」「締切 23:47:12」
