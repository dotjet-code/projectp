# Figma プロンプト 02 — トップページ

**前提**: `01-style-guide.md` のトークン・コンポーネント・ルールに完全準拠すること。STAGE モード基調、最下部の1ブロックのみ RACE モード（トンネル遷移）。

---

## 画面の目的

訪問者が 10 秒以内に以下 4 点を掴めること:

1. いま何節のどこ（開幕 / 中盤 / 終盤 / 特番）
2. 上位 6 (PLAYER) と 下位 6 (PIT) の現在地
3. 次の"事件"（ライブ、敗者復活、締切）までの残時間
4. 何に参加できるか（予想 / 投票）

詩的ヒーローは置かない。最初のフォールドから数字と名前と残時間が見える。

---

## レイアウト（上から）

### 1. ヘッダー（sticky）

- 高さ `56px`、背景 `--bg` に `1px --rule` 下線
- 左: Project P ロゴ（小。既存踏襲、ただし gradient は外し、黒1色）
- 中央ナビ: 本日の出走 / 暫定順位 / 予想 / 敗者復活 / 結果
- 右: 会員ログインまたはマイページ
- ロゴはテキストのみ（ブランドマークは左端に小さな競艇旗のピクトグラム 16×16）

### 2. 節タイムライン帯（sticky の直下、常設）

- 高さ `48px`、背景 `#FFFFFF`、下に `1px --rule`
- 横長に 4 節を等分で並べる:
  - 開幕 → 中盤 → 終盤 → 月末特番
- 現在位置を `--signal-red` の丸でマーク、通過済みは黒塗り、未到達は白抜き
- 各節直下に小キャプション（日付）
- 右端に現在節名 + 残時間（例: 「終盤 / 締切まで 2日 7時間 22分」）欧文数字フォント

### 3. 一面見出し（紙面トップ）— ポスター級インパクト

新聞一面 × 昭和アングラポスターの掛け合わせ。**左に縦長大判写真、右に活字祭り**の二分割。高さはデスクトップで `640–780px` を確保し、画面の 1.5–2 画面ぶんを一面が支配する。

#### 左 2/5: 縦長大判写真（被写体ポスター）

- 事件の主役メンバー 1 名を**全身 or 腰上の大判切抜き**で縦長 `400×640px` 以上
- デュオトーン処理（`--signal-ink` + `--bg`）+ 粒子ノイズ。元の色は全て消す
- 写真の上に**号艇色の斜め帯** (幅 `80px`、角度 `-12°`、3/4 透過) が上下を貫通
- 写真の下部または中央に**ゼッケン (01〜12) を彫込み風で 280pt**（写真と mix-blend-mode: difference）
- 写真の縁は**荒い切抜き**（なめらかなPNGトリミングではなく、鋸歯状のマスク）
- 写真の右端と見出し領域の間に `3px --signal-ink` の縦ケイ線

#### 右 3/5: 多段見出し組版

4 段で構成 (Style Guide §多段重ね組版 準拠):

1. 赤スタンプ `[速報]` 14pt（等級に応じて `[激震]` `[号外]` にも切替）
2. 超大見出し 1 行目（和文 72pt 黒、句読点で切る: 「首位、」）
3. 超大見出し 2 行目（和文 88pt 黒、右に 40px オフセット、2px 赤版ズレ: 「陥落。」）
4. サブテキスト 18pt 本文書体、32字程度

右上隅に**朱肉判子** (例: 事件等級 S なら「激震」) を −6° 傾けて押す。

#### 背景

- 紙面地色 `--bg`
- 左右セクションの境界上に**赤ペンの丸印**が 1 個、注目の数字を囲むように配置（手書き赤ペン痕、Style Guide §5 準拠）
- 左下隅に手書きコメント「※差 8pt / 昨日比 ▼3」（`Klee One` 赤）

下端に `6px --signal-red` の強調罫線（旧 3px を太く）。

#### 等級フォールバック時の絵づくり

事件なし (等級 E) の日は写真を暫定首位のポートレートに固定、大見出しは「**静かな一日。**」+ 手書き赤コメント「※大きな動きなし」に差替。寂しく見えないよう、背景に斜めストライプを 15% で薄く敷く。

#### 署名級モーション

- 初訪時: 篆刻 P が中央でドン演出 → その後、大見出しの 3 段目が右からスライドイン + 赤版が 2px ズレて 200ms で整列
- 日付が変わった時: 紙吹雪を 1.2 秒だけ撒く（派手すぎず、赤黒 12 枚）

### 3-b. 今日のひと言（煽りオノマトペ帯）

一面と次セクションの間に、**全幅の黒ベタ帯** (高さ `120px`) を挟む。

- 中央に A1 ゴシック B で「**激闘。**」「**沸騰。**」など 1 語、白抜き 96pt
- 右端に小さく日付と "Stage XX Day YY"
- 左端に出走表マーク（1-6 の号艇色タイル）

オノマトペは事件等級と連動:
- S: 「激震。」「ウォォ。」
- A/B: 「接戦。」「沸騰。」
- C: 「波乱。」「気配。」
- D/E: 「静寂。」「待て。」

#### 見出し自動選定の等級システム

サーバー側ロジックで直近 24 時間の事象を下記優先順で評価し、最上位等級をその日の大見出しに採用する。同等級が複数なら直近時刻を優先。

| 等級 | 事象 | 見出しテンプレ |
|---|---|---|
| S | 首位交代 | 「首位、交代。」「〇〇、陥落。」 |
| A | PLAYER/PIT ライン跨ぎの順位変動 | 「〇〇、PLAYER 圏外。」「〇〇、PIT 脱出。」 |
| B | 3 ポイント以内の接戦発生 | 「〇位 〇〇、差 Xpt。」 |
| C | 連勝/連敗の更新 | 「〇〇、N 節連続 上昇。」 |
| D | ライブ告知 / 敗者復活告知 | 「今夜、敗者復活戦。」 |
| E (フォールバック) | 事件なし | 「**静かな一日。**」+ 暫定首位インタビュー誘導 |

- 左 2/3: 超大見出し（和文 72pt / 欧文 88pt）
- 見出し下に 1 行サブテキスト（18pt / 32 字程度）: 定数テンプレ + データ差し込み
- 右 1/3: 紙面風「日付スタンプ」+「第 XX 節」+ 等級に応じた赤/黄バッジ（S/A = 速報赤、C/D = 告知黄）
- 下に `3px --signal-red` の強調罫線

**フォールバック (等級E) の見え方**: 「静かな一日。」の下に "暫定首位インタビュー" 小カードを添え、空白を"もったいぶり"に変換する。

印刷ズレモーション: ページロード時のみ赤版が 2px ズレて 200ms で整列。

### 4. 本日の出走（メンバー出走表）

セクション見出し: 「**本日の出走**」（中見出し 32pt、左に縦 `3px --signal-red` バー）

横スクロールではなく**縦積みの出走表**。12 行、上位 6 行と下位 6 行の間に**太い罫線 + 「—— PASS LINE ——」**（M-1の復活枠線、競艇の本命ライン）。

各行のカラム（左→右）:

| col | 内容 |
|---|---|
| ゼッケン (40px) | 永続番号 01–12 |
| 号艇 or PIT プレート (48px) | PLAYER = 号艇色×番号 / PIT = グレー×前節順位 |
| 顔写真 (56px 正方、デュオトーン処理) | 丸トリミング禁止 |
| 順位 (40pt 大数字) | 現在の暫定順位。左に `▲1` / `▼2` / `−` |
| 氏名 (18pt 和文太字) + ロール + ライブ中（赤点 6px、点滅は `prefers-reduced-motion` で停止）|
| スコアバー 3 段 (バズ / 配信 / 収支) | 右端に数値 |
| 今節ポイント合計 (32pt 欧文数字、tabular-nums) | 右揃え |
| 直近の動き (12pt キャプション) | 例: 「3 時間前 バズ +42」 |
| 「詳細 →」(文字リンク) | |

**PASS LINE の演出**:
- 上下に `3px --signal-red` の太罫線
- 中央に 8pt のラベル「— PASS LINE 6位以上が PLAYER —」
- この罫線にだけ斜めストライプ背景を薄く敷く

行の背景は基本 `--bg`、ホバーで `#FFFFFF` + 号艇色の左枠線 `3px`。カードではなく表。

### 5. 暫定順位グラフ（節内推移）

セクション見出し: 「**節内推移**」

横軸=日付、縦軸=順位（上=1位）。デフォルトは**注目5本表示**（首位 / 陥落中首位候補 / 最大上昇 / 最大下降 / ライン跨ぎ候補）。残り7本は灰色の背景線として薄く描画。

- 初期表示: 注目5本（それぞれ号艇色または PIT グレー）、残7本 `#D5CFC0` 20% opacity
- フィルタ: 「全表示」「注目のみ」「PLAYER のみ」「PIT のみ」の4ボタン
- PASS LINE は横に `3px --signal-red` の点線、左端に「6 位 / 7 位」ラベル。節途中は現在の PASS ポイントを実線で補助（「**今この瞬間、6 位は Xpt**」）
- ホバーで該当行の氏名・ポイント・順位ラベル
- 凡例は上部にゼッケン + 氏名で並べる（色だけでなくゼッケン番号で識別）

**目的**: 12 本描いて「情報過多だが熱い」のではなく、5 本で「**物語を選んで見せる**」。

### 6. 逆転ライン（メタ情報ブロック）

セクション見出しは**全幅の黒ベタ帯** (Style Guide §ベタ塗り面積) に白抜きで「**逆転ライン**」48pt。左上に赤スタンプ、右上に出走表マーク。

小カード 3 連（ケイ線 1px 区切り、角丸・影なし）:

- **最小点差**: 「6 位 〇〇 と 7 位 〇〇、差 **3pt**」（差の数字のみ 64pt 赤ベタ四角の中に白抜き）
- **連続上昇**: 「〇〇、**4 節連続**で順位アップ」（4 の部分に手書き赤ペンの丸印）
- **因縁**: 「先節 〇〇 に 11pt 差で敗北、**今節リベンジ中**」（右下に「差し」判子スタンプ −4° 傾け）

どれも「動詞 + 数字 + 固有名詞」の叩きつけ型コピー。カード背景は白ベタ、薄い粒子ノイズのみ。

### 7. ハーフタイム（節中盤のみ表示）

**表示条件**: 節の経過日数が全期間の 40%〜60% の範囲内のときのみ。条件外はこのブロックごと非表示（空白にしない、上下のセクションが詰まる）。

`--signal-yellow` と黒の斜めストライプで枠。見出し「**ハーフタイム中間順位**」。TOP 3 のゼッケン + 誌面処理済み肖像（正方 `120×120`、丸にしない、顔単体で映えないようデュオトーン + 粒子）。下に各人の前半の伸び率。

### 8. 敗者復活ゾーン導入（RACE モードへのトンネル入口）

ここが**紙面ページ随一のキャッチー**。

- ここから下はインクが流し込まれるように暗転 (Style Guide §署名級モーション「インク流し込みトンネル」)。高さ `400px` の遷移レンジ内でのみ。
- 罫線は黒 → `--signal-yellow` の残光に変化。黄の残光は画面横方向に 200ms かけて走り抜ける
- ブロック内部（暗転後）:
  - 画面幅いっぱいの斜め黄黒ストライプ帯が上端に `40px`
  - 中央に**ポスター見出し 160pt** で「**PIT、上げろ。**」(Anton Black or 貂明朝B、白、2–3px 赤版ズレ + 1px 青版ズレの**3版ズレ**でアングラ感)
  - 見出しの背後に薄く「**敗者復活戦**」を超大 240pt、10% opacity、画面幅に広がる（見出しと重ねる）
  - 中央右に**朱肉判子**「**待て**」 −12°、サイズ 120pt（締切前の意味）
  - 下に敗者復活枠 6 人の出走表。地色 `#111F37`、PIT プレートの縁に `--signal-yellow` 強調
  - CTA: 「**敗者復活戦を見る →**」太枠ボタン `--signal-red` 塗り、白文字、角丸 `4px`、押すと**篆刻 P の残光**が走る

### 9. ライブ情報ゾーン

セクション見出し: 「**本日のライブ** / **ライブ予定**」（2 サブブロック）

既存の情報を踏襲、ただし:
- カード表現を捨て、**ケイ線区切りのリスト**に変更
- 日付 + 会場 + 出演 + ステータスバッジ（投票受付中 / 開催予定 / 配信中）
- 「投票する →」は `--signal-red` の文字リンク + 右矢印（ボタン形状は最小限）

**ライブなし/予定なしのフォールバック**: 「**本日、配信なし。**」の小見出し + 「**次のライブまで X 日 HH:MM**」のカウントダウンのみを横長に 1 行で表示。空白ブロックを隠すのではなく、"次の事件を待つ看板"として機能させる。

### 10. 今月の因縁

「**先節の結果**」セクション。先月最終順位の上位 3 と落選 1 を横並びで小さく。「次節で再戦予告」的なコピーを添える。

### 11. フッター

シンプルに。ロゴ + リンク + 「**採点規程**」（透明性担保リンク） + コピーライト + 最終データ更新時刻 (`JST 12:34:08` 形式)。

右端に競艇の**締切時刻看板**の雰囲気で、小さな電光風の赤数字カウンタを常時表示。表示切替ロジック:
- 節進行中: 「締切まで X 日 HH:MM」
- 節確定〜次節開幕前: 「次節開幕まで X 日 HH:MM」
- 月末特番直前 (残 48h 以内): 「**月末特番まで** HH:MM:SS」+ 黄の強調罫線

## レスポンシブ

- ブレイクポイント: `768px` 以下でモバイル
- 出走表はモバイルで 2 行に折り返す:
  - 1 行目: 号艇 + 順位 + 氏名 + ポイント
  - 2 行目: スコアバー 3 段（細く横並び）
- 節タイムラインはモバイルでも 4 節表示を維持（ラベルを日付のみに短縮）
- 大見出しはモバイルで 48pt まで縮小

---

## マイクロインタラクション

- 節タイムラインの赤丸は 2 秒周期で薄い脈動（勝負モード外では半呼吸）
- 順位が更新された行は 600ms 間、左端の号艇色バーが厚く光る（1 回限り）
- スコアバーはマウントで左から伸びる 400ms
- ハーフタイム・敗者復活ゾーンの斜めストライプは静止（動かさない）
- すべて `prefers-reduced-motion: reduce` で無効化

## 実装メモ（デザイン制約）

- トンネル遷移（STAGE→RACE 暗転）は`IntersectionObserver`ベース or CSS `animation-timeline: view()`。長大なスクロール連動ではなく、**敗者復活ブロック手前 400px の範囲内でのみ**暗転させる。ページ全体が重くなるのを避ける。
- 節内推移グラフは D3 / recharts 等ではなく**静的SVG + CSS 変数**で十分。12本の曲線は事前計算してパスを出力。
- 一面大見出しの印刷ズレは `mix-blend-mode: multiply` + 赤版レイヤの transform、画像ではなく CSS で。
